これは、4つのエンティティで構成される単純な "supermarket" NGSI-LD データモデルの説明です:

  • Product - ストアで販売されている物
  • Shelf - ストア用家具のユニット。製品は棚にあります
  • StockOrder - 1つ以上のストアで販売されているアイテムの一覧のエントリ
  • Store - 物を売る建物

このチュートリアルで定義されている各エンティティに関するヒューマン・リーダブルな詳細情報は、リンクをクリックして見つけることができます。

@context の読み方 - 構文トークンとキーワード

JSON-LD 1.0 仕様 で定義されているように、 JSON-LD は NGSI-LD で使用されるいくつかの構文トークンとキーワードを指定します。チュートリアルの NGSI-LD データモデルの理解を容易にするために、主なキーワードを以下にリストします:

  • @context によって定義された Context (コンテキスト)
    JSON-LD ドキュメント全体で使用される短縮名を定義するために 使用されます
    詳細情報: セクション 5.1 .

  • @id で定義された Node identifiers (ノード識別子)
    定義されているもの (things) を一意に識別するために使用されます
    詳細情報: セクション 5.3.

  • @value で定義された Values (値)
    特定の プロパティ (property) に関連付けられたデータを指定するために使用されます
    詳細情報: セクション 6.4.

  • @type (typed values) にで定義された Data types (データ型)
    ノード (node) のデータ型 または 型付き値 (typed value)を設定するために使用されます
    セクション 6.4で説明しています

  • @vocab によって定義されたデフォルトの Vocabulary (語彙)
    共通のプレフィックス IRI で @type のプロパティと値を展開するために使用されます
    詳細情報: セクション 6.2.

  • @graph で定義された Named Graphs 名前付きグラフ
    JSON-LD graph グラフに明示的にラベルを付けるために使用されます
    詳細情報: セクション 6.13.

JSON-LD および NSGI-LD のすべての Keys (キー), keywords (キーワード)、および values (値) は大文字と小文字が区別されることに注意してください。

NGSI-LD と JSON-LD のリレーションシップ

NGSI-LD 仕様は NGSI-LD はリンクト・データをシリアル化する JSON ベースのフォーマットの JSON-LD に基づいていると規定されています。JSON-LD の @context は、 文字列として提供される用語を URI として指定された概念にマッピングするために使用されます。Core NGSI-LD (JSON-LD) @context は、以下を含む JSON-LD @context として定義されます:

  • NGSI-LD 情報モデルで定義された主要な概念を一意に表すために必要なコア用語
  • API 関連のデータ型を定義するすべてのメンバを一意に表すために必要な用語

NGSI-LD 準拠のすべての実装は、コンテキスト情報の処理または生成時に暗黙的に存在する Core @context の使用をサポートします。ただし、NGSI-LD 要素をレンダリングするときは、Core @context を常に明示的に参照する必要があります。これにより、必要に応じて、JSON-LD プロセッサは API 実装によって提供される JSON-LD ドキュメントを適切に展開できます。

完全な core NGSI-LD @context は、次の場所で公開されています: http://uri.etsi.org/ngsi-ld/v1/ngsi-ld-core-context.jsonld
